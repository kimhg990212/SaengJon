## 2. 백엔드 구조 전면 개편

### 2-1. 설정 모듈 (`backend/app/config.py`) 단순화

- **변경 전:** `pydantic-settings`의 `BaseSettings` 클래스 + `lru_cache` 패턴
- **변경 후:** `python-dotenv` + `os.getenv()` 직접 로드 방식으로 경량화
- NTS_API_KEY 누락 시 서버 기동 단계에서 즉시 `RuntimeError` 발생하도록 처리
- 관리 환경변수: `NTS_API_KEY`, `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `CORS_ORIGINS`

### 2-2. 삭제된 파일

| 파일 | 삭제 이유 |
|------|-----------|
| `backend/.env.example` | 제거 (직접 .env로 관리) |
| `backend/app/database.py` | SQLAlchemy ORM 방식 → psycopg2 직접 연결로 교체 |
| `backend/app/models/business.py` | ORM 모델 불필요 |
| `backend/app/models/__init__.py` | 동일 |
| `backend/app/routers/health.py` | health 라우터 → main.py 내 `GET /` 로 흡수 |
| `backend/app/schemas/common.py` | 공통 스키마 단순화로 제거 |
| `backend/app/services/__init__.py` | 미사용 |
| `backend/app/utils/__init__.py` | 미사용 |

### 2-3. 새로 추가된 파일

#### `backend/app/db.py` — DB 커넥션 풀
- `psycopg2.pool.SimpleConnectionPool` 사용 (최소 1 / 최대 10 연결)
- `get_conn()` / `release_conn()` 헬퍼 함수 제공

#### `backend/app/apis/nts_api.py` — 국세청 API 클라이언트
- 공공데이터포털 사업자등록 상태조회 API 연동
- 하이픈 없는 10자리 사업자번호를 POST로 조회
- 실패 시 `NtsApiError` 예외 발생

#### `backend/app/routers/business.py` — 사업자 조회 라우터
- `POST /api/business/status` 엔드포인트
- 24시간 캐시: DB에 저장된 데이터가 24시간 이내면 재조회 없이 반환 (`cached: true`)
- 캐시 miss 시 국세청 API 호출 → DB upsert → 응답 반환
- `psycopg2`로 직접 SQL 실행 (`ON CONFLICT DO UPDATE`)

#### `backend/app/schemas/business.py` — Pydantic 스키마
- `BusinessStatusRequest`: 사업자번호 검증 (10자리 숫자, 하이픈 자동 제거)
- `BusinessStatusResponse`: `b_no`, `b_stt`, `b_stt_cd`, `tax_type`, `end_dt`, `cached` 필드

### 2-4. `backend/requirements.txt` 변경

| 제거 | 추가 |
|------|------|
| `sqlalchemy` | `python-dotenv` |
| `pydantic-settings` | `requests` |
| | `pydantic` |

### 2-5. `backend/app/main.py` 변경
- `health` 라우터 제거 → `business` 라우터 등록
- `GET /` 루트 엔드포인트를 인라인으로 추가 (`{"status": "ok"}`)
- `settings.cors_origins` → `CORS_ORIGINS` 직접 참조

## 전체 변경 요약

1. **모노레포 구조 확립**: 프론트엔드 파일 전체를 `frontend/`로 이동, `backend/`와 명확히 분리
2. **백엔드 경량화**: ORM/pydantic-settings 제거 → psycopg2 직접 연결 + dotenv 단순 설정
3. **핵심 기능 추가**: 국세청 사업자 상태조회 API 연동 + 24시간 DB 캐시 레이어 구현
